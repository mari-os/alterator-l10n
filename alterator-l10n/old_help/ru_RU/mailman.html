Сервер электронных рассылок

Чем больше участников вовлечено в электронную переписку, тем труднее поддерживать список адресатов. Зачастую эту работу выполняет сервер электронных рассылок. С его помощью можно создавать списки и управлять ими через веб-интерфейс. Подписчики могут изменять параметры личной подписки, включать и отключать доставку, координатор списка — менять общие настройки, добавлять и удалять подписчиков.

Каждому списку назначается адрес веб-сайта, где он может быть настроен, и адрес почтового сервера (или домена), через который будут отправляться письма. Эти адреса могут быть указаны вручную либо определяться автоматически.

Из списка установленных почтовых серверов следует выбрать используемый MTA, а из списка языков — язык сайта. Язык веб-интерфейса для каждого списка можно будет указать дополнительно. Создайте главную рассылку сайта и укажите её имя. В неё предлагается писать по техническим вопросам.

Наряду с паролем администратора каждого из списков можно использовать один общий пароль администратора. Для создания новых списков через веб-интерфейс — пароль создателя списков. Задайте их на странице настроек.

Иногда бывает нужно поддерживать на одном сервере несколько виртуальных доменов и приписанных к ним списков рассылок. Добавьте необходимые домены или удалите ненужные.

Сразу после установки сервера он может оказаться не запущен. Пройдите по ссылке запустить, остановить или перезапустить службу и запустите его. (Там же можно узнать его текущее состояние.)
