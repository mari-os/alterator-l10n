Mailing list server

The more participants involved in the e-mail, the more difficult to list them. Often the mailing list server does the job. You can create and manage mailing lists with it via the web. Subscribers are able to modify their subscription options and toggle delivery, list administator — to change common list settings, add or remove subscribers.

Each list has an URL host, where it can be customized, and an Email host (or domain), through which messages will be sent. These addresses can be either entered manually or determined automatically.

From the list of installed mail servers you should choose the Site MTA, and from the list of languages — the Site language. Language for each list can be specified further. Create the default site list and enter it's name. It is proposed to write on technical matters.

Along with the list administrator passwords, you can use a common administrator password. To create new lists via web-interface — a list creator password. Set them in the settings.

Sometimes, it is useful to keep multiple virtual domains with mailing lists assigned to them on a single host. Add the necessary domains or remove unnecessary.

Immediately after installing mailman, it may not be running. Please follow start, stop or restart service link and run it. (There you can see its current state as well.)
