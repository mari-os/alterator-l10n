DHCP server

DHCP (Dynamic Host Configuration Protocol) - protokol brugt af klienten til at få automatisk en IP-adresse fra reserveret adresseområde og også nogle nyttige oplysninger om lokalt netværk (DNS-server, søgedomæne, standard gateway).

Typisk er den opnåede adresse gyldig i en begrænset periode og kan være genbrugt senere til en anden klient.

Nogle gange er det nyttigt at give samme IP adresse hver gang klienten opretter forbindelse til netværket. I dette tilfælde kan serveren genkende klienten ved hjælp af hardwareadressen på netværksadapteren (MAC-adresse). Du kan tilføje en sådan klient med en Statisk adresse.

Dynamisk DNS

DNS (Domain Name System) - protokol for at etablere et forhold mellem menneskevenlige symbolske navne og IP adresser.

I nogle distributioner kan du finde en DNS server i par med DHCP server. Denne server fungerer automatisk uden yderligere konfiguration og understøtter DNS-zone med navn lig med serverens domænenavn (se modulet "Domæne"). Denne server fungerer kun med statisk (ikke DHCP eller Zeroconf) konfigurerede netværksgrænseflader.

I dette tilfælde:

  • Dynamisk tildelt adresse vil automatisk registreres i DNS med navn svarende til klientcomputerens navn.
  • Tabel med statisk adresse vil automatisk blive knyttet til DNS (hvis computernavnet er angivet)
  • DNS server og dets domæne sendes automatisk til alle netværksklienter.

Opmærksomhed: DNS-serverens database vil blive nulstillet, hvis du ændrer domænenavn eller ændre en statisk IP-adresse. Liste over statiske DHCP-adresser vil også blive fjernet i dette tilfælde.
