Розподіл дисків

Якщо немає особливих побажань, варто використати автоматичний розподіл нерозміченого або всього доступного простору.

Для уникнення ненавмисної втрати даних перед застосуванням змін слід відключити USB flash та переносні вінчестери, які не використано при встановлені.

Ручний розподіл

До цього способу слід вдаватися досвідченим користувачам Linux і в будь-якому випадку бажано мати резервну копію всіх цінних даних на випадок помилки або збою.

Рекомендується зробити хоча б три розділи:

  • swap — розділ підкачки, звичайний розмір 1–2x RAM;
  • / — кореневий розділ, на якому розташовано систему;
  • /home — домашній розділ, на якому розташовано дані користувачів.

Для основних розділів рекомендується використовувати файлову систему ext3.

Типи файлових систем

ОС Linux підтримує різні файлові системи, які мають різні переваги та недоліки.

  • ext2: нежурнальована файлова система; відносно проста у відновленні, але потребує відносно тривалої перевірки цілісності після збою живлення або ядра. Може бути використана для /boot або readonly-розділів.
  • ext3: файлова система по умовчанню; журнальована та достатньо надійна, має середню продуктивність.
  • xfs: високопродуктивна журнальована файлова система, яка має більш високу чутливість до збоїв. Рекомендується для активно використовуваних файлових систем при умові стабільного живлення та якісної апаратної частини.
  • jfs: журнальована файлова система, яка має підтримку перекодування імен файлів (iocharset).
  • reiserfs: швидка журнальована файлова система; більш не розробляється; у окремих випадках можливе руйнування з невідновлюваною втратою даних. Не рекомендується для зберігання цінної інформації.

Файлові системи FAT/FAT32/NTFS підтримуються у встановленій системі, але не для встановлення на них Linux.

LVM/RAID

Для використання цих опцій потрібно створити розділи відповідного типу, потім обрати потрібний пункт.
