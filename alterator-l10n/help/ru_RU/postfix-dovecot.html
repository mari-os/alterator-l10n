Почтовый сервер

Этот модуль предназначен для настройки сервера SMTP, который отправляет сообщения по электронной почте другим пользователям на сервере и в Интернете, и сервера POP3/IMAP, который предоставляет пользователям доступ к электронной почте на сервере.

Сервер SMTP

Если вы хотите использовать сервер SMTP для отправки электронных писем, установите флажок «Включить службу SMTP».

Сервер SMTP может работать в двух режимах:

  • Посредник — пересылать исходящую почту на указанный сервер. При выборе этого режима будет показано поле адреса принимающего сервера, на который будет пересылаться вся почта.
  • Сервер — доставлять почту самостоятельно.

На домены, указанные в поле «Список доменов» почта принимается всегда. По умолчанию указывается домен сервера.

Для отправки и получения писем вы можете задать следующие ограничения:

Максимальный размер сообщения (в мегабайтах) — максимальный размер отправляемого или получаемого письма.

Разрешить аутентификацию — отправлять во внешнюю сеть только при указании правильного имени пользователя и пароля, заведённого на этом сервере. Отправка писем пользователям на этом же сервере может быть произведена также и без аутентификации. Отправители из сетей, заданных в поле «Внутренние сети», могут отправлять почту во внешнюю сеть без аутентификации.

Принимать незашифрованные пароли только через безопасные соединения — передача пользователям паролей только через шифрованные схемы (GSSAPI) или шифрованные соединения (SSL или TLS). При передаче пароля открытым текстом (PLAIN, LOGIN) через небезопасное соединение (отключено SSL и TLS) возникнет ошибка.

Помечать спам — задавать в принимаемых сообщениях вероятность спама по правилам сервера SMTP.

Фильтровать отправителей — принимать почту только из существующих доменов, не перечисленных в «чёрном» списке сервера xbl.spamhaus.org. Домены, IP-адреса которых указаны в поле «Внутренние сети» с маской подсети, имеют приоритет перед списком. Также разрешено передавать почту авторизованным пользователям.

Фильтровать получателей — запретить отправлять почту на указанные адреса электронной почты и домены, перечисляемые через пробел.

Сервер POP3/IMAP

Если вы хотите использовать сервер POP3/IMAP для доступа пользователей к электронной почте на сервере, установите флажок «Включить службу POP3/IMAP».

Для доступа к службам POP3 (порт 110) и IMAP (порт 143) пользователь должен включить в своём почтовом клиенте аутентификацию и указать своё имя и пароль. При использовании шифрованных соединений SSL и TLS порты будут другие.

Параметр «Принимать незашифрованные пароли только через безопасные соединения» разрешает пользователям передавать пароли только через шифрованные схемы (GSSAPI) или шифрованные соединения (SSL или TLS). При передаче пароля открытым текстом (PLAIN, LOGIN) через небезопасное соединение (отключено SSL и TLS) возникнет ошибка.
