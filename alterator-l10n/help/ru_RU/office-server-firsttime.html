Первоначальная настройка системы

Система развёрнута, для завершения процесса установки вам необходимо указать некоторые начальные параметры.

Прежде всего выберите язык интерфейса.

Каждый сервер обслуживает один домен - группу компьютеров одной сети, имеющую единый центр и использующую единые базы данных для различных сетевых служб. Имя домена может состоять только из малых латинских букв, а также символов '-','.' и '_'.

В сервере ALT Linux единую информационную систему обеспечивают cледующие службы:

  • Kerberos - центр аутентификации пользователей домена. Kerberos удобен тем, что позволяет не вводить имя учётной записи и пароль каждый раз при использовании сетевых служб и ресурсов. Например, таким образом обеспечивается прозрачная аутентификация при доступе к сетевым папкам предоставляемых сервером. Обеспечивается это выдачей так называемых билетов. Билеты выдаются на определённый срок, поэтому для корректного функционирования Kerberos необходима точная синхронизация времени между всеми машинами домена.
  • LDAP - база пользователей, групп пользователей и данных Kerberos.
  • DNS - база компьютеров в сети. Позволяет обращаться к другому компьютеру домена, используя его символическое имя, а не IP-адрес. DNS сервер работает в паре с сервером DHCP, и соотвествие между IP-адресами и именами комьютеров настраивается автоматически.

У каждая из вышеперечисленных служб имеются свои требования к именованию домена, поэтому введённое вами имя будет автоматически преобразовано для каждой из них. Например, если было выбрано имя "my.domain.org", то:

  • В DNS имя так и будет выглядеть как "my.domain.org";
  • В Kerberos имя будет приведено к верхнему регистру "MY.DOMAIN.ORG";
  • В LDAP имя будет приведено к виду "dc=my,dc=domain,dc=org.

Имя домена - важный параметр. Смена имени домена в установленной и настроенной системе может привести к удалению и созданию заново единых баз данных домена, поэтому желательно сразу задать правильное значение этого параметра

Ещё один важный параметр - пароль администратора (учётная запись root). Без этого пароля вы не сможете зайти в систему c консоли и в web-интерфейс центра управления системы.

Указание внешних сетевых интерфейсов позволит правильно настроить брандмауэр. Дальнейшая настройка может быть произведена в модуле "Брандмауер"
